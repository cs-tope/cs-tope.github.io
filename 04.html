AI Games

Harder to solve than toy problems (8 queens, 8 puzzle)

Large number of possible actions (branching factor). For example, in chess, the branching factor is about 35 in average.

Deeper search trees. In chess, one player makes around 50 moves.

Pruning

Games require the ability to make a decision even when calculating the optimal decision is impractical or impossible in real-time

Use heuristic evaluation functions (utility function).

Ignore portions of the search tree that make no difference to the final choice (nodes with bad heuristic values)

Game Representation

Initial State $S_0$ - how the game is set up at the start

Player(S) - defines which player takes a move.

Actions(S) - the set of legal moves in the current state

Result(S,A) - the resulting state after a move (transition model)

Terminal-Test(S) - returns true when game is over

Utility(S,P) - defines a numeric value to a terminal state s for a player p

Minimax Algorithm

Two players: Min and Max

Max makes an initial move, and a move for every response by Min

Minimax Algorithm

		
function MINIMAX-DECISION(state) returns an action
     return arg max MIN-VALUE(RESULT(state,a))

function MAX-VALUE(state) returns a utility value
     if TERMINAL-TEST(state) then return UTILITY(state)
     v <- -inf
     for each a in ACTIONS(state) do
          v <- MAX(v,MIN-VALUE(RESULT(s,a))
     return  v

function MIN-VALUE(state) returns a utility value
     if TERMINAL-TEST(state) then return UTILITY(state)
     v <- inf
     for each a in ACTIONS(state) do
          v <- MIN(v,MAX-VALUE(RESULT(s,a))
     return  v
